Database Comparison

SQL

  • Structured data model with fixed schema
  • Uses SQL for querying and manipulating data
  • Supports ACID transactions
  • Vertical scalability (scale up)
  • Strong consistency

NoSQL

  • Flexible data model with dynamic schema
  • Uses non-SQL for querying and manipulating data (e.g. JSON, XML)
  • Supports eventual consistency
  • Horizontal scalability (scale out)
  • Eventual consistency
